## StarUML 是什么？一句话说明白

## StarUML 教程：核心功能速览

### 拖拽式建模，不用背命令

打开 StarUML 以后，左侧工具栏有各种图元素——类、接口、关联、依赖、继承线……直接拖到画布上就能用。和 PlantUML 那种"用代码写图"的方式不同，StarUML 是完全可视化的拖拽操作，上手门槛低很多。

双击一个类图元，可以直接编辑类名、属性和方法。可见性用符号标注：`+` 是 public，`-` 是 private，`#` 是 protected。这套规则是 UML 标准，学一次，所有工具通用。

### 模型驱动开发，数据存为 JSON

这是 StarUML 和普通"画图工具"最本质的区别。StarUML 保存的不只是一张图，而是一个**模型**——数据以 JSON 格式存储，包含类、关系、约束等完整的语义信息。

有了这个 JSON，你可以通过命令行（CLI）调用用户自定义模板，自动生成 Java、C#、C++ 等语言的骨架代码。改了模型，代码框架也能跟着更新。这就是所谓的"模型驱动开发"（MDA），架构设计和代码开发不再是两条平行线。

### 插件系统，Java/Python 代码来回转换

StarUML 有活跃的开源插件社区，GitHub 上有大量现成的扩展，覆盖 Java 逆向工程、Python 代码生成、Spring 框架支持等场景。

安装插件不复杂：菜单 → Extensions → Extension Manager → 搜索安装。装完重启，功能就生效了。

### 异步模型验证

每次保存或打开模型文件，StarUML 会在后台自动跑一遍验证规则，检查模型逻辑有没有矛盾或遗漏，然后把问题列出来。不会打断你正在做的操作，但问题绝不会被悄悄漏掉。

### 导出格式齐全

图表画好了，能导出高 DPI 的 PNG/JPEG，也能生成 PDF 打印文件，还能一键发布成 HTML 静态站。发给不装 StarUML 的人看？直接丢 HTML 链接就行。

## StarUML 常用图表类型速查

学建模的时候，常常不知道什么场景用什么图。这里列个简明对照：

| 图表类型 | 用途 | 适用场景 |
|---|---|---|
| 类图（Class Diagram） | 描述类的结构和关系 | 面向对象系统设计 |
| 用例图（Use Case Diagram） | 描述系统功能与用户交互 | 需求分析阶段 |
| 序列图（Sequence Diagram） | 描述对象之间的消息时序 | 接口设计、流程梳理 |
| 状态图（State Diagram） | 描述对象状态变化 | 订单状态机、工作流设计 |
| 活动图（Activity Diagram） | 描述业务流程或算法步骤 | 业务逻辑说明 |
| 组件图（Component Diagram） | 描述系统组件架构 | 微服务、模块化设计 |
| ERD（实体关系图） | 描述数据库表结构和关系 | 数据库设计 |
| C4 模型 | 分层描述系统架构 | 向不同层级受众说明系统结构 |

## 常见问题（FAQ）

**Q：StarUML 支持中文界面吗？**

不支持。StarUML 目前只有英文界面。但操作逻辑并不复杂，菜单项都是常见英文单词，上手一两天就能熟悉。画出来的图里，类名、注释等内容是可以写中文的。

**Q：一个激活码可以用在几台电脑上？**

标准授权支持 1 台设备同时激活。换电脑时需要先在原设备执行反激活操作，然后再在新设备激活，授权可以随机器迁移。

**Q：买了之后能免费升级新版本吗？**

在数码荔枝购买的版本支持子版本免费更新（如 7.0 升级到 7.x）。如果未来官方发布大版本（如从 V7 升到 V8），通常需要以折扣价单独购买升级授权。如果开发商调整升级政策，以官方最新说明为准。

**Q：30 天试用期结束后，没买的话数据会丢失吗？**

不会。StarUML 的试用逻辑是：不激活也能用，只是部分功能有提示。你之前保存的模型文件（`.mdj` 格式）不受影响，可以继续打开和编辑，直到你决定是否购买。

**Q：StarUML 和 draw.io 有什么区别，哪个适合我？**

draw.io 是"画图工具"，画出来的是图形，不包含语义模型数据。StarUML 是"建模工具"，存储的是带有类关系、约束信息的完整模型，支持代码生成和逆向工程。

如果你只是偶尔画个流程图给别人看，draw.io 足够了。如果你需要做严肃的软件架构设计、做代码和模型的来回同步，StarUML 才是对的工具。
